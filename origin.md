# 关于Mac用户无法打开APP的建议方法

因为苹果官方不鼓励用户安装第三方 APP，所以 Mac 用户往往会因为系统权限问题，打不开非appstore下载的App，具体原因和解决方案比较多，一般根据报错信息自行搜索都能解决。常见的几个情况有：

# 1.提示"文件已损坏，无法打开"

## macOS Mojave 10.14及以下系统（即使不是这个版本的也可以试试）

打开终端，输入:

```
sudo spctl --master-disable
```

输入开机密码并回车，返回打开app即可

## macOS Catalina 10.15系统（即使不是这个版本的也可以试试）

打开终端，输入:

```
sudo xattr -rd com.apple.quarantine 文件路径
```

回车，返回打开app即可

## macOS Catalina 10.15.4及以上系统（即使不是这个版本的也可以试试）

打开终端，输入:

```
sudo codesign --force --deep --sign -文件路径
```

回车，返回打开app即可

请优先根据系统版本尝试对应方法，如果无效，有一定几率尝试非对应版本方法，也能成功。

# 2.提示"您没有权限打开应用"或者"应用程序不能打开"

解决方案：

右键app，点「显示包内容」，依次进Contents->MacOS，会看到图标显示不正常的执行文件，

打开终端，输入:

```
chmod +x 执行文件路径
```

（加号不可省略）

回车，会看到执行文件图标变化，返回打开app即可

注：文件路径可通过直接将文件拽入终端获得，空格不可省略

# 此外

无法打开的情况下，请都尝试使用右键-打开，而非左键双击打开。右键打开 = 管理员权限打开。
